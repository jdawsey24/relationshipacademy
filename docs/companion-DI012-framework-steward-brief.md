## How to read this brief

Each situation below poses a single framework question: **is the phase wrong, is the competency wrong, or is neither — should it stay at the developmental-task level?** For each I state the validated phase, the steward-proposed competency, the exact nature of the conflict, the relevant MKB v2.1 competencies (cited by ID), and a recommendation. No competency is invented; no framework construct is modified. The final decision is yours.

**Standing constraint:** the situation's *phase* was validated in the registry audit. Where I recommend "change the competency," I am asserting the phase is sound and only the competency label was off. Where I flag a possible phase question, I say so explicitly.

## DI-012.1 — RS-0020

- **Situation:** "I Want to Create Better Dating Boundaries"
- **Validated phase / task:** **Exploration / Discernment**
- **Steward-proposed competency:** *Boundary Communication*
- **The conflict:** "Boundary Communication" is a real canonical competency, but it lives in a **different phase**. MKB defines it as **`COM-EXCL-003` Boundary Communication — Exclusivity, Communication.** There is no "Boundary Communication" in Exploration.
- **Relevant MKB competencies:**
  - Named: `COM-EXCL-003` **Boundary Communication** (Exclusivity) — the phase conflict.
  - Same-phase (Exploration): `ROL-EXPL-005` **Boundaries** (Role Functioning); `PHY-EXPL-001` **Boundaries** (Physical Intimacy).
- **Framework read:** Setting boundaries *while dating* is squarely discernment work — it is Exploration. The phase is sound. Exploration already carries boundary competencies; the steward simply named the *communication-of-boundaries* variant that the framework only introduces at Exclusivity.
- **Recommendation: Change the competency (keep the phase).** Assign `ROL-EXPL-005` **Boundaries** as primary (general relationship boundaries), or `PHY-EXPL-001` **Boundaries** if the intent is physical/sexual boundaries. **Do not** move the situation to Exclusivity — dating boundaries are not intentional-investment work.
- **Confidence:** High.

## DI-012.2 — RS-0027

- **Situation:** "I Am More Invested Than They Are"
- **Validated phase / task:** **Exploration / Discernment**
- **Steward-proposed competency:** *Pacing*
- **The conflict:** This is **not a phase conflict** — it is a **non-existent construct.** MKB v2.1 contains **no competency named "Pacing" in any phase.** So there is nothing to place; the label does not correspond to a framework competency.
- **Relevant MKB competencies (Exploration candidates):**
  - `COM-EXPL-006` **Reciprocity** (Communication) — matches the *investment-imbalance* framing directly.
  - `EMI-EXPL-001` **Gradual Self-Disclosure** (Emotional Intimacy) — matches the *pace-of-intimacy* framing.
  - `ROL-EXPL-004` **Reciprocity** (Role Functioning); `COM-EXPL-003` **Perspective Taking** (Communication) — secondary.
- **Framework read:** The situation is unambiguously early-dating discernment — Exploration is correct. The developmental work behind "more invested than they are" is recognizing **reciprocity** (mutual investment). "Pacing" was a descriptive shorthand, not a framework term.
- **Recommendation: Change the competency (keep the phase).** Assign `COM-EXPL-006` **Reciprocity** as primary. If the steward specifically meant the tempo of emotional opening rather than investment balance, `EMI-EXPL-001` **Gradual Self-Disclosure** is the alternative. **No phase change; no new competency.**
- **Confidence:** High on phase; the Reciprocity-vs-Gradual-Self-Disclosure choice is the steward's to confirm (intent-dependent).

## DI-012.3 — RS-0048

- **Situation:** "We Need to Discuss Sex Before Marriage"
- **Validated phase / task:** **Expansion / Integration**
- **Steward-proposed competency:** *Transition Communication*
- **The conflict:** "Transition Communication" is a real competency, but in a **different phase**: MKB defines it as **`COM-EXPR-002` Transition Communication — Expiration, Communication.** Expiration is the *ending/decline* phase; discussing sex before marriage is integration work, not relationship dissolution. Applying an Expiration competency here would mis-signal the developmental context.
- **Relevant MKB competencies:**
  - Named: `COM-EXPR-002` **Transition Communication** (Expiration) — the phase conflict.
  - Same-phase (Expansion): `COM-EXPN-001` **Shared Decision-Making** (Communication); `COM-EXPN-004` **Adaptive Communication** (Communication).
- **Framework read:** "Before marriage" places the couple in pre-marital integration — Expansion / Integration. Deciding together about sexual expectations before marriage is a **shared decision**. The phase is sound; the proposed competency belongs to the wrong end of the life cycle. (Note: "Sexual Communication" exists but only as `PHY-EXCL-003` in Exclusivity — the same cross-phase pattern, so not an Expansion option.)
- **Recommendation: Change the competency (keep the phase).** Assign `COM-EXPN-001` **Shared Decision-Making** as primary; `COM-EXPN-004` **Adaptive Communication** if the emphasis is communicating through a changing context. **Do not** move to Expiration.
- **Confidence:** High.
