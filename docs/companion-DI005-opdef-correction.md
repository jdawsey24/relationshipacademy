# DI-005 — Operational Definitions Manual §3.2 Correction

**Reference:** Data Integrity finding DI-005
**Date:** 2026-07-20
**Prepared by:** Systems Architect
**Applies to:** *RLC Operations Manual* (Operational Definitions Manual) — **Construct 3.2, Developmental Task**
**Nature:** Terminology correction to align the Operational Definitions Manual with the Theory Manual and MKB v2.1. **No framework change** — the developmental task itself is unchanged; only its label is corrected to the canonical term.

---

## Authority

- **Owner ruling:** the canonical Exclusivity developmental task is **"Intentional Investment."**
- **Theory Manual** governs meaning; **MKB v2.1** is the source of record — both use **"Intentional Investment"** (verified: MKB `04_Competencies` Exclusivity rows carry `Developmental Task = "Intentional Investment"`, single value).
- The Operational Definitions Manual §3.2 is the **only** canonical document still using the truncated form **"Investment."** Per the canonical hierarchy (Theory → Op-Def → implementation), the Op-Def text must be corrected to match.

## The correction

**Location:** Construct 3.2 — *Developmental Task* → **"Primary Developmental Tasks"** table → **Exclusivity** row.

| | Developmental Phase | Primary Developmental Task |
|---|---|---|
| — before — | Exclusivity | **Investment** |
| — after — | Exclusivity | **Intentional Investment** |

**Corrected table in full (for context):**

| Developmental Phase | Primary Developmental Task |
|---------------------|----------------------------|
| Exploration | Discernment |
| Exclusivity | **Intentional Investment** ← corrected |
| Expansion | Integration |
| Expiration | Acceptance |
| Recovery | Healing |
| Renewal | Reengagement |

That is the entire change: one cell.

## Scope confirmation (what NOT to change)

- This is the **only** instance of the standalone task term "Investment" in the manual (verified: exactly one occurrence outside of "Emotional Investment").
- **Do not change "Emotional Investment."** It appears twice in the manual as a **different construct** — a competency in the **Emotional Intimacy** domain — and is unrelated to the Exclusivity developmental task. It must remain "Emotional Investment."
- No other phase→task pairing changes; the other five are already correct.

## Verification after applying

- All six phase→task pairs in Construct 3.2 will match the Theory Manual and MKB v2.1 exactly.
- The term "Intentional Investment" (currently absent from the Op-Def manual) will appear once, in the Exclusivity row.
- No downstream implementation edits are required — the Companion architecture workbook, registries, and live data already use "Intentional Investment" (this correction closes the gap from the *other* direction, bringing the source manual into alignment).

## Recommended housekeeping

- Bump the Operational Definitions Manual version and note the correction in its change log: *"§3.2 Construct — Exclusivity developmental task relabeled 'Investment' → 'Intentional Investment' to align with Theory Manual and MKB v2.1 (DI-005)."*
- On applying, DI-005 moves from "Resolved (implementation)" to fully closed at the source.

---

**Summary:** one word, one cell — "Investment" → "Intentional Investment" in Construct 3.2's Primary Developmental Tasks table. Everything else in §3.2, and the two "Emotional Investment" mentions elsewhere, stay exactly as written. This is a labeling alignment, not a framework modification.
